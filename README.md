## 📌 Overview

This project was structured around one central question:

> *Why do some products fail despite strong technology, and how do you fix them?*

The project covered two parallel streams:

1. **RCA Frameworks** — learning and applying structured diagnostic tools (MECE, Fishbone, 5 Whys, PESTEL, SWOT, JTBD, AARRR, CIRCLES) to real product failures
2. **Applied Case Studies** — using those frameworks to produce three detailed strategy documents (Google Stadia teardown, Netflix India growth strategy, Lisners GTM plan) and one engagement RCA (Zomato)

---

## 🧠 Frameworks Covered

### Diagnostic Frameworks
| Framework | Purpose |
|---|---|
| **MECE** (Mutually Exclusive, Collectively Exhaustive) | Structure problem spaces without overlap or gaps |
| **Fishbone Diagram** (Ishikawa) | Identify all possible causes of a specific problem |
| **5 Whys** | Uncover root cause by iteratively asking "Why?" |
| **PESTEL** | Analyze macro-environmental factors (Political, Economic, Social, Technological, Environmental, Legal) |
| **SWOT** | Evaluate internal strengths/weaknesses and external opportunities/threats |
| **JTBD** (Jobs To Be Done) | Understand why customers "hire" a product functionally, emotionally, and socially |

### Growth & Engagement Frameworks
| Framework | Purpose |
|---|---|
| **AARRR** (Pirate Metrics) | Map user funnel: Acquisition → Activation → Retention → Revenue → Referral |
| **DAU/MAU Ratio** | Measure product stickiness; diagnose engagement vs superficial adoption |
| **CIRCLES Method** | Structured 7-step framework for product design decisions and trade-off evaluation |
| **4Ps / STP** | Marketing positioning and go-to-market strategy |

## 📂 Case Studies

### 1. Google Stadia — Product Teardown
**Type:** Root Cause Analysis · Published on [Medium](https://medium.com/@pmgroupiitk/product-teardown-google-stadia-da439d431584)

Google Stadia was a cloud gaming service launched in November 2019 and discontinued in January 2023 — despite Google's massive infrastructure advantage.

**Frameworks Applied:** MECE · Fishbone · SWOT · AARRR · PESTEL

**Key Findings:**
- Limited game library with no exclusive titles — users had no reason to switch from existing platforms
- Streaming quality fell short: most games upscaled from 1080p rather than true 4K, with significant input latency in fast-paced games
- Confusing subscription model: users still had to purchase individual games on top of the subscription fee
- Poor device support, missing features, and a confusing controller setup gave the platform an unfinished feel
- Google's reputation for discontinuing products (Google+, Inbox, Allo) eroded user trust from launch day

**Root Cause:** Weak content ecosystem and immature execution undermined user trust and adoption despite strong cloud infrastructure.

### 2. Netflix India — Growth Strategy
**Type:** Engagement Analysis + Feature Design + Marketing Plan

Netflix grew from **21.5M subscribers (2011)** to **277.6M (2024)** globally, but India remains underleveraged relative to its population size.

**Frameworks Applied:** DAU/MAU · AARRR · CIRCLES · 4Ps · SWOT

**Engagement Gap Identified:** Low DAU/MAU ratio signalling episodic rather than habitual usage — users open Netflix occasionally, not daily.

**5 Proposed Engagement Features:**

| Feature | Rationale |
|---|---|
| Short-form content feed | Address shrinking attention spans; compete with Reels/Shorts |
| Watch Party | Drive social/shared viewing; increase session length |
| Daily Streaks | Gamify daily viewing habit formation |
| Live Events (sports/concerts) | Drive real-time engagement and urgency |
| Regional Drama Hub | Capture Hindi/regional language content demand in Tier 2/3 cities |

**Marketing Plan:** ₹21L/month acquisition strategy with channel-wise budget allocation across digital, influencer, and regional channels.

**Projected Impact:** **33–52K incremental daily active users**, exceeding the original **25K target**.

### 3. Lisners — Go-to-Market Strategy
**Type:** Competitive Benchmarking + GTM Design + KPI Framework

Lisners is a mental health and wellness app by Brightlanes Technologies (Delhi, 2020) focusing on Tier-2/3 cities — connecting users with professionals via GDPR-compliant audio/video consultations.

**Frameworks Applied:** SWOT · PESTEL · JTBD · STP · 4Ps

**Competitive Benchmarking (5 Rivals):**

| Competitor | Positioning |
|---|---|
| Wysa | AI-first, anonymous mental health support |
| YourDOST | Expert counselling, corporate partnerships |
| Lissun | Clinical focus, therapist network |
| Evolve | Self-help tools, CBT-based exercises |
| MANAS | Government-backed, free access |

**Lisners' Differentiation:** Focus on smaller cities, strong privacy, freemium model (₹500–1000/month), peer community forums.

**GTM Budget:** ₹10–15L across 6 categories (digital acquisition, influencer, college outreach, B2B, PR, retention)

**8 Measurable KPIs Defined:**

| KPI | Target |
|---|---|
| App Downloads (Month 1) | 10,000+ |
| User Conversion (Free → Paid) | 20% |
| 30-Day Retention | 60% |
| Session Completion Rate | 75% |
| NPS Score | 40+ |
| CAC | < ₹200 |
| LTV:CAC Ratio | > 3:1 |
| Referral Rate | 15% |

### 4. Zomato — Retention RCA
**Type:** Engagement Diagnosis using DAU/MAU + Cohort Analysis

**Problem Statement:** Zomato's acquisition is high but retention is falling — a classic DAU/MAU divergence pattern.

**Framework Applied:** 6-step PM Action Framework + AARRR + CIRCLES

**Diagnostic Questions Applied:**
- What are the common traits of retained vs churned users?
- Is the first-time experience confusing or underwhelming?
- Are notifications being ignored or causing friction?
- Are there bugs affecting repeat usage?
- Is the app valuable enough for frequent daily use?

**Root Cause Hypothesis:** Onboarding gaps and irrelevant notification strategy causing first-time users to try but not return.

**CIRCLES Method Applied:** Evaluated a product trade-off between adding friction (longer delivery ETA accuracy) vs removing friction (faster checkout) using the 7-step structured decision framework.

## 📊 Engagement Metrics Deep Dive

### DAU/MAU Ratio — Industry Benchmarks

| Product Type | DAU/MAU Ratio | Driver |
|---|---|---|
| Social Media (Instagram) | 50–70% | High habit-forming behavior |
| Messaging (WhatsApp) | 60–80% | Daily communication need |
| E-commerce (Amazon) | 15–30% | Event-based usage |
| Professional (LinkedIn) | 10–20% | Occasional usage |
| Streaming (Spotify/Netflix) | 30–50% | Entertainment-based patterns |

### How to Improve DAU/MAU

| Strategy | Mechanism |
|---|---|
| Improve onboarding | Ensure Day 1 engagement and value delivery |
| Smart notifications | Bring users back meaningfully without annoying them |
| Habit-forming hooks | Streaks, challenges, personalized recaps |
| Personalization | Tailored experience → more relevance → more use |
| Feature audits | Remove friction points blocking repeat use |
